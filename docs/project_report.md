# 北京师范大学学生绩点（GPA）预测项目报告
## 1. 项目背景与意义
学业绩点（GPA）是北京师范大学衡量学生学业水平、评价学习成效的核心量化指标，直接关联学生评奖评优、学业预警、推免升学、就业择业等关键环节。作为教育部直属的全国重点大学、“双一流”建设高校，北师大始终坚持拔尖创新人才培养与学生全面发展相结合，传统学业管理模式多以“事后统计分析”为主，难以提前识别学生学业风险、精准匹配帮扶资源，导致学业辅导工作的针对性和高效性有待提升。

本项目基于机器学习算法，结合北师大学业管理实际与学生行为特征，整合出勤次数、作业完成率等8个核心学业特征，构建高精度绩点预测模型，实现学期绩点的提前精准预测。项目成果可直接服务于北师大的学生管理与人才培养工作：
1. 提前识别学业风险学生，为辅导员开展个性化、前置化学业辅导提供数据支撑；
2. 贴合北师大推免、奖学金评定规则，为学业评价工作提供量化参考；
3. 分析影响学生学业的核心特征，为优化课程教学、提升课堂教学质量提供方向；
4. 为学生提供个性化学习规划指导，助力学业能力与综合素养双提升；
5. 完善北师大“数据驱动”的精准化学生管理体系，契合学校双一流建设的发展要求。

项目采用贴合北师大校情的模拟数据开发，代码模块化、无硬编码，可快速迁移至北师大真实脱敏学生数据场景，兼顾专业性、实用性与工程化落地能力。

## 2. 数据说明
### 2.1 数据来源
使用模拟生成的北京师范大学在校本科生数据，参考学校各专业学业行为特征，覆盖22个核心教学单位，保证数据与北师大校情、办学特色的贴合性与合理性。

### 2.2 数据维度与说明
结合北师大人才培养特点，设计8个特征变量和1个标签变量（绩点），特征选取兼顾学业评价与学生综合表现，所有特征取值范围贴合北师大本科生学业管理实际规则：

| 特征名称          | 特征类型 | 说明                     | 取值范围         | 特征作用               |
|-------------------|----------|--------------------------|--------------|------------------------|
| major             | 类别     | 所属教学单位             | 北师大22个核心教学单位 | 区分不同学科、专业的学业特点 |
| gender            | 类别     | 性别（0=女，1=男）       | 0/1          | 基础人口统计特征       |
| attendance        | 数值     | 学期必修课出勤次数       | 20-32        | 反映课堂参与度，贴合北师大课堂考勤管理要求 |
| homework_completion | 数值   | 作业完成率（含实践作业） | 0.6-1.0      | 核心学业特征，反映课后学习与实践任务完成情况 |
| lib_borrow        | 数值     | 周均图书馆借阅量         | 0-10         | 反映自主学习积极性，贴合北师大浓厚的学术氛围 |
| club_participation | 类别    | 是否参加社团/志愿活动（0=否，1=是）| 0/1          | 反映课余时间分配，兼顾学生实践能力培养需求 |
| class_interaction | 数值     | 课堂互动次数（提问/回答） | 0-20         | 核心特征，反映课堂投入度与参与积极性 |
| exam_score        | 数值     | 期中/期末测试成绩        | 60-100       | 核心特征，反映学科知识与专业知识掌握程度 |
| gpa               | 数值     | 学期绩点（预测标签）     | 1.0-4.0      | 模型预测目标，贴合北师大绩点评定标准 |

**注**：所属教学单位包含北师大教育学部、文学院、数学科学学院、人工智能学院、化学学院、心理学部等，覆盖学校核心院系与特色专业。

### 2.3 数据规模与划分
- 总样本量：10000条（覆盖22个教学单位，样本分布均匀，适配各专业比例）；
- 训练集：8000条（占比80%，用于模型训练）；
- 测试集：2000条（占比20%，用于模型评估，与训练集无重叠）；
- 随机种子：固定为42，保证实验结果可复现，符合学术研究规范。

### 2.4 数据生成逻辑
绩点生成基于北师大学业规律设计强线性相关逻辑，贴合学校实际教学场景：
1. 作业完成率（含实践作业）和测试成绩为核心影响特征，权重最高，贴合学校“理论+实践”的培养要求；
2. 课堂互动次数、出勤次数为次要核心特征，正向影响绩点，突出课堂参与的重要性；
3. 社团/志愿活动参与为适度负向特征（合理范围内不影响学业，过度参与则产生影响），贴合学生平衡实践与学业的实际需求；
4. 加入少量随机噪声（0~0.2），模拟实际学业中的不确定因素，提升数据真实性；
5. 最终绩点通过`.clip`强制限制在1.0-4.0，与北师大绩点评定的合理范围保持一致。

## 3. 方法与流程
项目核心流程遵循“数据生成-数据预处理-特征工程-模型训练-模型评估-预测应用-可视化GUI开发”的机器学习工程化流程，全流程自动化实现，一键运行即可完成全部操作，贴合北师大实际使用场景。

### 3.1 数据预处理
针对模拟数据与北师大真实学生数据的共性问题，设计完善的预处理逻辑，保证数据质量：
1. 缺失值处理：数值特征用中位数填充（抗异常值干扰），类别特征用众数填充，适配高校学生数据的缺失特点；
2. 异常值处理：基于北师大学业管理业务规则限制特征值域，超出范围的异常值替换为区间边界值；
3. 数据类型转换：将性别、社团/志愿活动参与等类别特征转换为整数类型，适配机器学习算法输入要求；
4. 数据校验：添加核心列校验与教学单位合法性校验，防止特征缺失、无效数据导致模型训练失败。

### 3.2 特征工程
结合北师大特征数据的类型特点，构建标准化特征预处理流水线，差异化处理数值与类别特征，避免数据泄露：
1. 数值特征：采用`StandardScaler`标准化，消除量纲差异，公式：$X_{scaled} = \frac{X - \mu}{\sigma}$（$\mu$为均值，$\sigma$为标准差）；
2. 类别特征：采用`OneHotEncoder`独热编码，设置`drop="first"`避免虚拟变量共线性，适配不同教学单位、性别的类别特征；
3. 流水线构建：用`ColumnTransformer`整合标准化与独热编码，实现特征的批量、自动化处理，方便后续模型复用与真实数据迁移。

### 3.3 模型选择与训练
选择3个经典回归模型对比训练，覆盖线性模型、树模型、集成学习，兼顾模型可解释性与预测性能，适配北师大绩点预测的业务需求：
1. 线性回归（基线模型）：采用普通最小二乘法拟合，训练速度快、可解释性强，便于分析特征与绩点的线性关联关系；
2. 决策树：通过网格搜索（GridSearchCV）调优`max_depth`、`min_samples_split`参数，5折交叉验证，提升模型泛化能力；
3. 随机森林（集成学习）：200棵决策树组成，设置`max_depth=12`防止过拟合，利用多核CPU加速训练，提升预测稳定性。

训练好的最优模型与特征预处理流水线通过`joblib`保存至`models/`目录，支持后续GUI界面快速调用与二次训练。

### 3.4 模型评估
#### 3.4.1 评估指标
选择3个核心回归评估指标，全面评估模型预测性能，贴合绩点预测的业务评价需求：
1. MAE（平均绝对误差）：衡量预测绩点与真实绩点的平均偏差，值越小精度越高，直观反映预测误差；
2. RMSE（均方根误差）：对大误差敏感，反映模型预测的稳定性，避免出现极端预测偏差；
3. R²（决定系数）：衡量模型对绩点变化的解释力，取值范围[0,1]，越接近1说明模型拟合效果越好。

#### 3.4.2 可视化评估
自动生成3张专业评估图表并保存至`results/`目录，直观展示模型性能，便于非技术人员理解：
1. 模型性能对比图：柱状图展示3个模型的R²指标，快速识别最优模型；
2. 特征重要性Top10图：横向柱状图识别影响北师大学生绩点的核心特征；
3. 真实vs预测绩点散点图：直观反映模型预测精度，添加完美预测线（y=x），清晰展示预测偏差。

### 3.5 可视化GUI开发
基于`tkinter`开发适配北师大使用场景的可视化GUI界面，作为项目前端展示层，降低非技术人员的使用门槛，核心功能如下：
1. 特征可视化输入：教学单位、性别、社团/志愿活动参与采用下拉/单选框（选项贴合北师大实际），数值特征带范围提示和类型校验，防止无效输入；
2. 一键绩点预测：调用训练好的最优模型，快速输出预测绩点与学业评级（贴合北师大评奖评优、学业预警规则），弹窗提示预测结果；
3. 模型评估图表查看：支持3张专业评估图表的实时查看，图表自动缩放适配界面，方便快速了解模型性能；
4. 智能错误提示：模型缺失、输入值超出范围、图表缺失时给出明确指引，提升使用体验。

GUI界面与项目核心预测逻辑完全解耦，可独立运行，适配北师大辅导员、学业管理人员的日常使用习惯。

## 4. 实验结果与分析
### 4.1 模型性能评估结果
在测试集上的模型评估指标（结果可复现，贴合北师大校情数据特点）：

| 模型         | MAE（平均绩点误差） | RMSE（均方根误差） | R²（决定系数） |
|--------------|---------------------|--------------------|----------------|
| 线性回归     | 0.1620              | 0.2004             | 0.9012         |
| 决策树       | 0.2194              | 0.2720             | 0.8180         |
| 随机森林     | 0.1763              | 0.2175             | 0.8836         |

### 4.2 结果分析
结合北师大校情与数据特点，对模型性能结果分析如下：
1. 线性回归模型表现最优（R²=0.9012，MAE=0.1620），说明北师大学生绩点与所选8个特征之间存在较强的线性相关关系，符合学校“重基础、重课堂、重实践”的培养规律；
2. 决策树模型表现较差，即使经过参数调优仍未达到理想效果，原因是数据集无复杂非线性关系，且决策树易对北师大不同教学单位的样本产生过拟合；
3. 随机森林模型表现良好但略逊于线性回归，因集成学习的优势在复杂非线性数据集中更明显，本项目数据贴合北师大实际学业规律，线性特征占主导；
4. 所有模型的MAE均小于0.23，预测绩点的平均偏差控制在0.2以内，预测精度完全满足北师大精准化学业管理、学业预警的实际业务需求。

### 4.3 特征重要性分析
基于随机森林模型的特征重要性排序（Top5），贴合北师大学生学习特点：
1. 测试成绩：核心影响因素，反映学生对学科基础与专业知识的掌握程度；
2. 作业完成率：次核心因素，实践作业与理论作业的完成质量直接决定学业掌握程度；
3. 课堂互动次数：第三核心因素，课堂投入度与参与积极性正向影响绩点；
4. 出勤次数：基础影响因素，北师大课堂教学重知识传递与思维训练，缺勤直接影响核心知识的学习效果；
5. 周均图书馆借阅量：反映学生的自主学习积极性，贴合北师大浓厚的学术氛围，正向影响绩点。

### 4.4 典型案例预测结果
结合北师大不同专业、不同学业水平的学生特点，选取2个典型案例的预测结果，贴合学校实际：
1. 优秀学生：人工智能学院，出勤30次，作业完成率0.98，课堂互动18次，测试成绩95分→预测绩点4.0（优秀等级，贴合北师大推免、特等奖学金评定标准）；
2. 学业预警学生：文学院，出勤22次，作业完成率0.65，课堂互动2次，测试成绩62分→预测绩点1.0（学业预警等级，需辅导员一对一介入辅导）。

预测结果与北师大实际学业水平评价预期高度契合，模型能准确识别不同专业、不同学业水平的学生，满足精准化学业管理的需求。

## 5. 应用与优化建议
### 5.1 北师大专属应用场景
本项目成果可直接落地应用于北京师范大学的学生管理、教学改革与人才培养工作，核心应用场景如下：
1. 学业预警与帮扶：对预测绩点低于1.5的学生开展前置化、一对一学业辅导，重点针对作业完成率、课堂互动不足的问题制定帮扶计划，提升学业辅导效率；
2. 评奖评优参考：将预测绩点高于3.5的学生纳入推免、奖学金候选名单，结合北师大现行评定规则，为量化评价提供数据支撑；
